# 共通ルール

## 命名規則

| 対象 | ルール | 例 |
|------|--------|-----|
| トップレベルファイル | 英語・小文字 | `spec.md`, `deploy.md` |
| サブディレクトリ | 英語・小文字 | `spec/`, `design/` |
| サブドキュメント | 英語・小文字・ハイフン区切り | `design/character.md`, `design/ui-redesign.md` |
| 番号プレフィックス | **使わない** | README.md の目次で順番を表現 |
| ファイル内容 | **日本語** | タイトル・本文はすべて日本語 |

### サブドキュメントパターン

親ファイル + サブディレクトリで概要と詳細を分離する。

```
{topic}.md    ← 概要 + サブドキュメントへのリンク一覧
{topic}/      ← 詳細ファイル群
```

- サブドキュメントが2つ以上あるとき分割。1つならフラット
- 1ファイルで収まるならサブディレクトリ不要
